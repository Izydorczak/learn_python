Differences Between the Shell and a File


Demo of the Shell


How do I get to the shell?

We'll use a program called IDLE. It comes with Python.
To run IDLE, open the command line and type IDLE.
For Windows, hit the Windows key, type "cmd", and hit enter.
For Mac, open the Terminal application in your Applications Folder.

Here's what IDLE's Python shell looks like. You should see this open on your own computer.
There are other applications to use the shell besides IDLE, but for now we'll use IDLE.


What do I do with the shell?

The shell interacts directly with Python. You type one instruction, Python executes that one instruction. Things go back and forth, one at a time. You, Python. You, Python.
Try typing a math expression.

You should see Python execute the math expression and print out the result.

Python will echo back whatever you asked it to print.

What if we remove the word "print" from our instruction?

Python will echo back whatever you typed. But notice the difference between this response and our previous instruction with the word "print".
Learning to write code well involves noticing differences between two approaches, no matter how small they are. What might look like a tiny difference to a new eye could be a really big difference.

Demo of a File


How do I get to a file?

In IDLE's menu, go to "File" -> "New File".
Now you should just see a blank file, without the triple carrot ">>>" that showed up on the next line of the shell.

What do I do with the file?
Type a single instruction in the file and save it. I used the name "firstFile.py", and my single instruction is shown in the code sample below. Make sure to get the ".py" extension. I usually save to the desktop when I'm testing something like this. For now, don't worry about the line that says "This Gist brought to you by gist-it" in the code sample above. Don't type it out. Your file should only have one line, to print the word "hi".

Now, in the IDLE menu, click "Run" -> "Run Module".

Your file will run, and any "print" instruction will print to the shell window.

This is a key difference, that was not nearly as obvious when our code was in the shell. We won't dwell on it for now-- it's enough to know that you need to include the word "print" each time we want output from running a file.

Key Differences

So now you've run some Python code in the shell and in a file. The key differences:

  • The shell goes back and forth, one line at a time, like a conversation.
  • A file allows you to type multiple instructions and then run them all at the same time.
  • Some of the output that you see when typing at the shell is not visible when the same code is run from a file.

It helps to think of the shell as a conversation, where you get a lot of feedback right away about how your words are received, but you can only express your ideas in small chunks. A file is more like an essay, where you can get out a lot of related ideas but you won't know how the reader responds until the essay is complete.
