Here's a small dictionary.

Notice the syntax: there are colons between each pair of items. In a dictionary, everything is stored in a pair.
But a dictionary is more than just pairs of items-- the first member of the pair is a "key", and the second is a "value".
In this example, "Wendy" is a unique key, and its value is "Jansson". We can look up any value in the dictionary using its key. We can also add a new key-value pair.
Just like we used a bracket to select a single ordered element from a list, we use a bracket to select a single value for a key, or store a new value for a key.

The first value will just be replaced by the second value. This means that the keys must be unique! We can't have two "Joe" keys in the same dictionary with different values.

Notice that we can't control the order of iteration for a dictionary, like we could for a list.
Dictionaries are a hashmap, which store key-value pairs. There is no inherent order, just the unique keys to store their values.
